Architecture

Two-Workflow System

WORKFLOW 1: Content Generator (dGQwIMyZImbYwucC)
[Onboarding Form] → [Parse] → [Save to Baserow] → [Scrape Website]
                                    ↓
                         Status: awaiting_notes


WORKFLOW 2: Build Notes Trigger (4BaNEGdjuP5EsLUy)
[Build Notes Form] → [Fetch Onboarding] → [Call OpenClaw]
    → [Process Response] → [Search Images] → [Upload to GHL Media]
    → [Update GHL Values] → [Update Baserow]
                                    ↓
                         Status: generated



UTILITY: Website Scraper (bbqKaTdP2idMG7OS)
[Webhook] → [Fetch HTML] → [Parse Content] → [Return]



UTILITY: Image Uploader (WrMcZYiQtYHXi9yG)
[Webhook] → [Download from MinIO] → [Upload to GHL] → [Return HTTPS URLs]

Data Precedence

  1. BUILD NOTES (highest) — Fulfillment team corrections
  2. ONBOARDING FORM — Client-provided data
  3. SCRAPED DATA — Existing website (supplementary)

Trigger

NOT triggered by onboarding form submission.

Content generation is triggered by the Build Notes Form in Slate Ops. This allows the Fulfillment Specialist to:

  • Review form data first

  • Add image selection preferences

  • Provide corrections/direction


Why Two-Step?

  1. Form submission saves data + scrapes website
  2. Human reviews, selects images, adds notes
  3. Build Notes form triggers actual generation
  4. Better quality control, fewer regenerations needed

Image Library

Source: MinIO bucket slate-image-library/{trade}/ Metadata: Baserow table 947 Total: 719 images across 37+ trades

Image Selection Order

  1. Background images (3) — hero_suitable=true only
  2. Backup hero options (11) — variety for testing
  3. Service page heroes (12) — 1 per service page
  4. Area page heroes (14) — 1 per area page
  5. About image (1) — any trade image
  6. Service images (36) — 3 per service
  7. Area images (28) — 2 per area
  8. Gallery (24) — remaining stock

Image Upload Flow

MinIO stores HTTP URLs → Image Uploader workflow downloads → Uploads to GHL Media Library → Returns HTTPS URLs → Custom values get googleapis.com URLs

Performance: ~0.78s per image

AI Observations

The agent outputs ai_observations containing:

  • Logo suggestion (Ideogram prompt for logo creation)

  • Color rationale (why these colors were chosen)

  • Assumptions made (inferred from data)

  • Data quality notes (missing/incomplete info)


Example:
Logo Suggestion: Military-inspired design with sergeant stripes incorporating paint brush.
Colors: dark green (#094026) primary, gold (#ffbc00) accents.



Ideogram Prompt: "Professional logo for Sergeant Painters, veteran-owned painting company,
military sergeant stripes chevron design incorporating paint brush, dark forest green and
gold colors, clean modern design, vector style, white background --ar 1:1"



Color Rationale: Military green conveys veteran ownership, discipline, trust. Gold accent
provides premium feel and excellent CTA visibility.

Selective Regeneration

Purpose: Fix specific fields without regenerating everything. Form URL: https://slate-tools.slatesystemsllc.workers.dev Workflow ID: jDGPTsr1OOLjsfxR

How It Works

  1. Enter Location ID
  2. Check fields to regenerate
  3. Add optional feedback/direction
  4. Submit

Smart Context

The AI sees existing content (for consistency) but EXCLUDES the field being changed (to avoid anchoring bias).

CategoryContext Rule
Core IdentityAlways include (unless being changed)
ServicesInclude 3 others if changing any service
AreasInclude 3 others if changing any area
StatsInclude all others (they work as set)
FAQsInclude 2 others for tone matching
BlogsInclude 1 complete blog as reference

When to Use

  • QA reveals placeholder or poor content
  • Client requests specific changes
  • Services/areas need refinement
  • Blog posts need different tone

Cost

Each regeneration costs API credits. Regenerate only what's needed.

Integration with Client Build

The Content Generator is invoked during Task 7 of the client build process:

  1. Client submits onboarding form → Workflow 1 runs → Status: awaiting_notes
  2. Fulfillment reviews form, fills Build Notes form
  3. Build Notes submission → Workflow 2 runs → Content generated
  4. Human check: Verify custom values in GHL Settings → Custom Values
  5. If issues: Use Selective Regeneration form
See: fulfillment/client-build.md for full build process
